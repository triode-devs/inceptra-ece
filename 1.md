# Quiz Backend — Complete API Documentation

> **Base URL (local dev):** `http://10.156.151.119:8787`  
> **Base URL (production):** `https://your-worker-name.your-subdomain.workers.dev`  
> All requests/responses use **JSON** unless noted.  
> All timestamps are **ISO 8601** (e.g. `2024-03-03T12:00:00.000Z`).  
> Token lifetime: **24 hours**.

---

## Standard Response Envelopes

```json
// Success
{ "success": true, "data": { ... } }

// Error
{ "success": false, "error": "Human readable message", "code": "ERROR_CODE" }
```

---

## Image URLs

> Every image field (e.g. `question_image`, `option_a_image`, `answer_image`) in API responses is **either `null` or a full public URL**:
> ```
> https://your-worker.workers.dev/images/questions/<uuid>/question_image_1234567890.jpg
> ```
> Use this URL directly in an `<img src="..." />` tag. No additional fetching or key lookup needed.

```json
  }
```


```bash
curl http://localhost:8787/admin/questions/q-uuid \
  -H "Authorization: Bearer <token>"
```

---

### `POST /admin/question-sets/:setId/questions`

Create a question under a set. Supports **JSON** (text-only) or **multipart/form-data** (with images).

**Option A — JSON:**
```bash
curl -X POST http://localhost:8787/admin/question-sets/set-uuid/questions \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "question_text": "What is 2+2?",
    "option_a_text": "3",
    "option_b_text": "4",
    "option_c_text": "5",
    "option_d_text": "6",
    "correct_option": "B",
    "marks": 4,
    "negative_marks": 1
  }'
```

**Option B — Multipart (with question image):**
```bash
curl -X POST http://localhost:8787/admin/question-sets/set-uuid/questions \
  -H "Authorization: Bearer <token>" \
  -F "question_text=What is shown?" \
  -F "correct_option=A" \
  -F "option_a_text=Capacitor" \
  -F "option_b_text=Resistor" \
  -F "option_c_text=Inductor" \
  -F "option_d_text=Diode" \
  -F "marks=4" \
  -F "negative_marks=1" \
  -F "question_image=@/path/to/image.jpg"
```

**Response `201`:** Created question object with `correct_option` and R2 keys (use the separate image upload endpoints to get the public URLs back).

### `PATCH /admin/rounds/:id/promote`

Trigger top-N promotion. **All batches in the round must be `ended` first.**  
Tie-breaking: `total_marks DESC` → `correct_count DESC` → `submitted_at ASC`.

**Request Body:**
```json
{ "promote_top_n": 10 }
```

**Response `200`:**
```json
{
  "success": true,
  "data": {
    "promoted_count": 10,
    "promoted": [
      { "participant_id": "reg001", "name": "Alice", "total_marks": 75 }
    ]
  }
}
```

```bash
curl -X PATCH http://localhost:8787/admin/rounds/round-uuid/promote \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"promote_top_n":10}'
```

### `POST /participant/room/answer`

Submit one answer. Can be called per question as the participant progresses. Can be re-submitted to change an answer (upserted).

> If the batch timer has expired when this is called, the batch is auto-ended before responding.

**Request Body:**
```json
{
  "batch_id": "batch-uuid",
  "question_id": "q-uuid",
  "selected_option": "B"
}
```

**Response `200`:**
```json
{
  "success": true,
  "data": { "is_correct": true, "marks_awarded": 4 }
}
```
> `marks_awarded` is **negative** if wrong and `negative_marks > 0`.

**Error `403` (timer expired):** `{ "success": false, "error": "Batch has ended. Cannot submit answers.", "code": "BATCH_ENDED" }`

```bash
curl -X POST http://localhost:8787/participant/room/answer \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"batch_id":"batch-uuid","question_id":"q-uuid","selected_option":"B"}'
```

## 12. Images (Public)

### `GET /images/:r2key`

**Auth required:** None — fully public.  
Serves the raw image bytes from R2 storage.

> The `:r2key` is the path from the image upload response, e.g. `questions/uuid/question_image_123.jpg`.  
> These keys are embedded directly into the question URLs in all API responses — you do not need to call this manually.

```bash
curl https://your-worker.workers.dev/images/questions/uuid/question_image_123.jpg
```

Returns the image binary with headers:
- `Content-Type: image/jpeg` (or actual type)
- `Cache-Control: public, max-age=31536000`

## Error Code Reference

| Code | HTTP | Meaning |
|---|---|---|
| `MISSING_FIELDS` | 400 | Required fields not provided in request body |
| `UNAUTHORIZED` | 401 | Missing, expired, or invalid Bearer token |
| `FORBIDDEN` | 403 | Authenticated but not allowed to perform this action |
| `NOT_FOUND` | 404 | Resource does not exist |
| `ALREADY_IN_ROUND` | 409 | Participant already joined a different batch in this round |
| `BATCH_ENDED` | 403 | Batch timer expired or batch manually ended by admin |
| `BATCH_NOT_STARTED` | 403 | Action requires batch to be in `started` state |
| `BATCH_NOT_PENDING` | 400 | Delete attempted on a non-pending batch |
| `ROUND_NOT_PENDING` | 400 | Update/delete attempted on a non-pending round |
| `WRONG_MODE` | 400 | Room mode mismatch (e.g. self_service endpoint on a room-mode room) |
| `INVALID_OPTION` | 400 | `selected_option` or option image key not one of `A`/`B`/`C`/`D` |
| `INVALID_MODE` | 400 | `mode` not one of `self_service` or `room` |
| `MISSING_FILE` | 400 | Image upload endpoint called without a file |
| `SERVER_ERROR` | 500 | Unexpected internal error |
